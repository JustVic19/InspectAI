Auth with Clerk

Clerk handles everything: sign up, sign in, sessions, user management. Three files and you have working auth across the entire app.

Why Clerk over NextAuth? Clerk gives you a hosted UI, user management dashboard, and webhook support out of the box. For a solo founder shipping fast, it saves days of work. Free up to 10,000 monthly active users.

2
Checkout route — creates the Stripe session
When the user clicks "Start free trial", your frontend calls this route. It creates a Stripe Checkout session and returns the URL to redirect to.
src/app/api/stripe/checkout/route.ts
ts
import { auth } from '@clerk/nextjs/server'
import { stripe } from '@/lib/stripe'
import { supabaseAdmin } from '@/lib/supabase'

export async function POST(req: Request) {
  const { userId } = auth()
  if (!userId) return Response.json({ error: 'Unauthorised' }, { status: 401 })

  const { priceId } = await req.json()

  // Get or create Stripe customer
  const { data: user } = await supabaseAdmin
    .from('users')
    .select('*')
    .eq('clerk_user_id', userId)
    .single()

  let customerId = user?.stripe_customer_id
  if (!customerId) {
    const customer = await stripe.customers.create({
      metadata: { clerk_user_id: userId }
    })
    customerId = customer.id
    await supabaseAdmin.from('users')
      .update({ stripe_customer_id: customerId })
      .eq('clerk_user_id', userId)
  }

  const session = await stripe.checkout.sessions.create({
    customer: customerId,
    mode: 'subscription',
    line_items: [{ price: priceId, quantity: 1 }],
    success_url: `${process.env.NEXT_PUBLIC_APP_URL}/dashboard?success=true`,
    cancel_url: `${process.env.NEXT_PUBLIC_APP_URL}/?canceled=true`,
    subscription_data: {
      trial_period_days: 7,
      metadata: { clerk_user_id: userId }
    }
  })

  return Response.json({ url: session.url })
}
3
Webhook route — unlocks access after payment
Stripe calls this route when subscription events happen. This is how your app knows someone has paid and should get access.
src/app/api/stripe/webhook/route.ts
ts
import { headers } from 'next/headers'
import { stripe } from '@/lib/stripe'
import { supabaseAdmin } from '@/lib/supabase'

export async function POST(req: Request) {
  const body = await req.text()
  const sig = headers().get('stripe-signature')!

  let event
  try {
    event = stripe.webhooks.constructEvent(
      body, sig, process.env.STRIPE_WEBHOOK_SECRET!
    )
  } catch (err) {
    return Response.json({ error: 'Webhook error' }, { status: 400 })
  }

  const subscription = event.data.object as any
  const clerkUserId = subscription.metadata?.clerk_user_id

  switch (event.type) {
    case 'customer.subscription.created':
    case 'customer.subscription.updated':
      await supabaseAdmin.from('users').update({
        subscription_status: subscription.status, // trialing | active
        plan: 'professional'
      }).eq('clerk_user_id', clerkUserId)
      break

    case 'customer.subscription.deleted':
      await supabaseAdmin.from('users').update({
        subscription_status: 'canceled',
        plan: 'free'
      }).eq('clerk_user_id', clerkUserId)
      break
  }

  return Response.json({ received: true })
}
Important: Stripe webhooks need the raw request body — not parsed JSON. The req.text() call above is deliberate. If you use req.json() the signature verification will fail.

Step 5

The AI Route

The Claude API call must live server-side — never in the browser. This route receives the checklist, builds the prompt, calls Claude, streams the response back, and saves the report to Supabase.

Why server-side? Your ANTHROPIC_API_KEY must never reach the browser. If you call Claude from a client component, your API key is exposed in network requests and anyone can steal it and run up your bill.
1
The streaming report generation route
This receives the checklist data, verifies the user has an active subscription, calls Claude with streaming, and saves the final report to Supabase.
src/app/api/generate-report/route.ts
ts
import Anthropic from '@anthropic-ai/sdk'
import { auth } from '@clerk/nextjs/server'
import { supabaseAdmin } from '@/lib/supabase'

const anthropic = new Anthropic()

export async function POST(req: Request) {
  const { userId } = auth()
  if (!userId) return Response.json({ error: 'Unauthorised' }, { status: 401 })

  // 1. Verify active subscription
  const { data: user } = await supabaseAdmin
    .from('users').select('subscription_status, plan, reports_used_this_month')
    .eq('clerk_user_id', userId).single()

  if (!['trialing', 'active'].includes(user?.subscription_status)) {
    return Response.json({ error: 'Subscription required' }, { status: 403 })
  }

  // 2. Check monthly report limit for starter plan
  if (user?.plan === 'starter' && user.reports_used_this_month >= 25) {
    return Response.json({ error: 'Monthly report limit reached' }, { status: 429 })
  }

  // 3. Get inspector profile for context
  const { checklistData, meta } = await req.json()
  const { data: profile } = await supabaseAdmin
    .from('inspector_profiles').select('*')
    .eq('clerk_user_id', userId).single()

  // 4. Build system prompt from profile
  const systemPrompt = `You are an expert home inspection report writer.
Inspector: ${profile.name}, ${profile.company}
License: ${profile.license_number} (${profile.state})
Style: ${profile.report_style}
Generate professional, clear reports. Be objective and actionable.`

  // 5. Stream the response
  const encoder = new TextEncoder()
  let fullReport = ''

  const stream = new ReadableStream({
    async start(controller) {
      const response = await anthropic.messages.stream({
        model: 'claude-sonnet-4-5',
        max_tokens: 2000,
        system: systemPrompt,
        messages: [{ role: 'user', content: buildUserPrompt(meta, checklistData, profile) }]
      })

      for await (const chunk of response) {
        if (chunk.type === 'content_block_delta' && chunk.delta.type === 'text_delta') {
          fullReport += chunk.delta.text
          controller.enqueue(encoder.encode(chunk.delta.text))
        }
      }

      // 6. Save completed report to Supabase
      await supabaseAdmin.from('reports').insert({
        clerk_user_id: userId,
        address: meta.address,
        client_name: meta.clientName,
        inspection_date: meta.date,
        checklist_data: checklistData,
        generated_report: fullReport
      })

      // 7. Increment report counter
      await supabaseAdmin.from('users').update({
        reports_used_this_month: (user.reports_used_this_month || 0) + 1
      }).eq('clerk_user_id', userId)

      controller.close()
    }
  })

  return new Response(stream, {
    headers: { 'Content-Type': 'text/plain; charset=utf-8' }
  })
}
2
Call the route from your React component
Replace the direct Claude API call in your existing component with a fetch to your own API route. Same streaming behaviour, but now it's secure.
src/components/ReportGenerator.tsx — replace the old callClaude()
tsx
async function generateReport(meta: any, checklistData: any) {
  const res = await fetch('/api/generate-report', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ meta, checklistData })
  })

  if (!res.ok) throw new Error(await res.text())

  // Stream the response into state
  const reader = res.body!.getReader()
  const decoder = new TextDecoder()
  let result = ''

  while (true) {
    const { done, value } = await reader.read()
    if (done) break
    result += decoder.decode(value, { stream: true })
    setOutput(result) // update state as it streams
  }
}
